沙箱环境运行 Claude Code 实践记录

背景

Claude Code 是 Anthropic 推出的 AI 编程 CLI 工具，支持在终端中直接进行代码生成、修改、调试等操作。本文记录了在美团沙箱环境中通过 CatClaw（内部 AI 助手）调用 Claude Code 进行开发的实践经验。

构建配置

  • 约定调用 cc 指令作为 Claude Code 的快捷命令
  • CatClaw 作为中间层，将用户指令转发给 Claude Code 执行

实践方式

整体流程：

  1. 在 CatClaw 对话框中用自然语言描述需求
  2. CatClaw 解析需求，生成 Claude Code 指令
  3. Claude Code 在沙箱环境中执行代码修改
  4. 将结果返回给 CatClaw 展示

收益

收益点 说明
知识库复用 CatClaw 本身会读取工程约束文档，相当于 Claude Code 作为子 Agent 执行代码任务，同时继承 CatClaw 的业务上下文
移动端操作 用 CatClaw 作为中转站，可以实现移动端操作 Claude Code 写代码
记忆系统 CatClaw 的记忆系统作为简易知识库，替 Claude Code 解析带业务含义、个人习惯的表达内容

局限性

问题 说明
耗时增加 多了一步 Agent 解析过程，相较于直接使用 Claude Code 耗时更长
信息遗漏风险 对 Claude Code 执行输出中的关键信息，比较依赖 CatClaw 的识别，可能遗漏一些重要内容
会话持久性 如果 Claude Code 没有安装在沙箱中，重启沙箱后 Claude Code 会话会丢失

适用场景

  • 适合：需要结合业务上下文生成代码、移动端触发代码任务、快速验证想法
  • 不适合：需要精准控制 Claude Code 行为的场景、对响应速度要求高的场景

小结

通过 CatClaw + Claude Code 的组合，可以在保留 CatClaw 业务上下文能力的同时，借助 Claude Code 的代码执行能力。这种”AI 套 AI”的架构在实践中是可行的，但需要接受额外的延迟和一定程度的信息损失。

对于追求极致效率的开发者，直接使用 Claude Code CLI 仍然是更优选择；而对于需要将 AI 编程能力与业务知识库深度结合的场景，这种混合架构有一定价值。
