QMScript

发表于 2025-05-06 分类于 笔记 ， 编程 阅读次数：

QMScript

这个名称是在按键精灵文件夹里看到的，姑且认为他的脚本语言就是认为这个名称（网上说是缝合了VBS、VB等等）

个人认为，按键精灵对于程序员来说并不是个很好用的语言，原因有以下几点：

  1. PC 版 UI、交互、IDE 什么的比较屎，你在写脚本的时候拖拽窗口会明显感觉到窗口像是在追着鼠标跑一样。。。
  2. 你想像其他高级语言一样自定义一个通用库，只能在内部库里面新建，而且调用方式也比较繁琐
  3. win10 会有缩放问题，如果你电脑不是 100% 的缩放，那么用抓抓选区域或者锁定句柄总会少一部分显示内容
  4. 自带库里面的识图容易抽风，除非是现截现用，否则会出现识不出来的情况
  5. 语言 bug 还是有挺多的，出现过用 Do While flag Loop 在进入下一次循环前 flag 变量值变了的情况（感觉很逆天，问题里会详细讲）
  6. 软件不维护了，最新的 PC 版本都是14年的。。。

但总的来说有个优点，就是相对来说比较方便（可能是我没用过其他类似的软件），能够很容易地模拟用户操作，一般都是用来搞游戏挂机刷副本之类的自动化操作。相较于用 Python 搓一个脚本来说，按键精灵肯定没那么高的灵活性，比如嵌入 OCR 或者 AI 识别之类的。

Do While … Loop 异常跳出循环

这代码原先是用雷电模拟器，搞我的勇者的钥匙副本挂机的。本来搞一个测试脚本复现的，但是发现怎么都复现不了，只好把这段代码全贴上来了。

其中主要的故障部分是在 70 行，在 Do While gaming 这层循环执行到 106 行给 gaming 赋值后理应正常跳出循环，但是他会跳转到第 70 行再次赋值（没错，我给70-111行每一行代码都穿插了 gaming 的输出调试了，就是会跳到那里）

然后我不信邪，就把前面空行都删了，以为是空行导致的 bug，结果还是不行，只能说十分逆天
