AI Agent 开发学习路径

整理自个人学习探索，系统梳理 AI Agent 开发所需的知识体系，从零到落地的完整路径。

一、需要掌握的基础能力

编程基础

  • 扎实的编程基础（Python / Node.js 为主流语言）
  • 熟悉异步编程和并发处理（Agent 任务天然并发）
  • 理解 LLM 的基本原理：Prompt Engineering、上下文窗口管理、Token 计算

框架与工具链

主流 Agent 框架（至少熟悉一两个）：

框架 定位
LangChain / LangGraph 最主流，生态最丰富
AutoGen 多 Agent 对话框架
CrewAI 角色驱动的 Agent 团队
OpenAI Assistants API 托管式 Agent 平台

新兴标准值得重点关注：

  • MCP（Model Context Protocol）：Anthropic 提出的工具调用标准协议
  • SKILL：工具调用的能力描述层

核心设计能力

Agent 开发的精髓在于系统设计，主要包括：

  • Tool 设计与封装：工具的颗粒度、参数设计、错误处理
  • 记忆系统设计：短期/长期记忆管理策略
  • 多步骤任务规划：ReAct、CoT、Plan-and-Execute 等范式
  • 多 Agent 协作架构：角色分工与信息流设计
  • 错误处理与重试：容错策略与降级方案

工程化能力

真正落地的 Agent 需要：

  • 可观测性：日志、Trace、评估体系
  • 流式输出处理：用户体验优化
  • 状态持久化：跨会话的上下文管理
  • 安全与权限控制：防止 Prompt Injection
  • 成本与延迟优化：Token 效率与响应速度

二、Agent 记忆系统

为什么需要记忆系统

LLM 本身是无状态的，每次对话从零开始。真正有用的 Agent 需要：

  • 跨会话积累知识
  • 记住用户偏好
  • 沉淀任务执行经验

记忆的三种形态

类型 说明 特点
Token-level 上下文窗口（Context） 直接但有容量上限，会话结束即消失
Parametric 微调进模型权重 持久但更新成本极高
External 向量库/数据库/文件系统 可读写、可增长，工程主流方向

从功能角度，记忆还可分为：

  • 用户记忆：偏好、画像
  • 任务记忆：执行过的案例、踩过的坑
  • 知识记忆：文档、规则

传统 RAG 的局限

传统 RAG 是平铺式向量存储，存在以下问题：

  • 丢失文档结构
  • 切片边界问题
  • 无法做全局判断
  • 检索是黑箱，无法理解完整语境
  • 无法实现”越用越聪明”的自我迭代

三、Plan（规划）——让 Agent 自主决策

Agent = LLM + Tools + Memory + Plan

Plan 让 Agent 从「被动应答」升级为「主动解决问题」。

常见规划策略对比

策略 适用场景 Token 消耗 核心优势
一次性计划 步骤固定、结果可预测 低 简单直接，执行快
动态计划 结果不确定、需灵活应变 中 适应性强，能处理异常
树形规划 多路径探索、方案选型 高 全局最优，可比较方案
树形 + 剪枝 大搜索空间、需控成本 可控 平衡质量与成本
ReAct 信息不完整、边做边学 中 最灵活，工程实践主流

选型建议：大多数场景用 ReAct 即可；步骤固定时用一次性计划；多方案选优时用树形规划 + 剪枝。

四、MCP（Model Context Protocol）

MCP 是 Anthropic 在 2024 年底提出并开源的协议标准，解决 AI 模型统一连接外部工具和数据源的问题，类比 USB 接口——一旦标准化，所有工具都可以即插即用。

架构三角色

MCP Server 能提供

  • Tools：可调用的函数（如查询数据库、发邮件）
  • Resources：可读取的数据（如文件内容、API 响应）
  • Prompts：提示词模板

五、Skill 与 MCP 的关系

核心区别：

  • MCP 是协议（通信层）：负责 Agent 和工具之间的标准化通信
  • Skill 是能力描述（认知层）：描述 Agent 能做什么、怎么做

类比：Skill 是大脑，MCP 是手脚。

最佳实践：

  • Skill 脚本负责业务逻辑的灵活编排
  • MCP Server负责单一职责的标准化工具执行

六、可观测性工具

工具 定位 特点
LangChain 开发框架 用来构建 Agent 的”积木”
LangSmith 调试平台 看 Agent 在干什么，与 LangChain 强绑定
Langfuse 观测平台 开源自部署，框架无关，数据安全

正式项目推荐自部署 Langfuse：不依赖特定框架，数据留存本地，可以追踪所有 LLM 调用的 Trace、Token 消耗、延迟和评估结果。

七、完整学习路径（四个阶段）

第一阶段（1-2 周）：地基夯实

目标：不依赖框架从零写出能调用外部工具的 Agent

  • 掌握 Node.js / Python 异步编程
  • 理解 Tool Calling（Function Calling）机制
  • 手写一个能调用外部 API 的简单 Agent

第二阶段（3-4 周）：框架深入

目标：掌握主流框架，实现 RAG 和记忆系统

  • 深入学习 LangChain / LangGraph
  • 实现 RAG（Retrieval-Augmented Generation）
  • 工程化记忆系统（向量库选型、检索策略）
  • Week 4-7：综合项目实践

第三阶段（3-4 周）：多 Agent 协作

目标：设计并实现多 Agent 协作系统

四种协作模式：

模式 说明
Supervisor 一个主 Agent 分发任务给子 Agent
Hierarchical 多层级的任务分解与上报
Peer-to-Peer Agent 之间平等通信与协商
Pipeline 线性流水线，每个 Agent 处理一个阶段

同时掌握 Agent 安全与防护：

  • Prompt Injection 防御
  • 沙箱隔离
  • 成本控制与限速

第四阶段（3-4 周）：工程化落地

目标：生产级 Agent 系统的工程化能力

  • 可观测性：Langfuse Trace、Token 追踪
  • 评估体系：LLM-as-Judge、自动化评测
  • 部署优化：流式输出、状态持久化、模型分级调用

学习路径总结

建议：跟着路径一步步做项目，不要只看文档——Agent 开发最重要的是在真实任务中踩坑和积累经验。
