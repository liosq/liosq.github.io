SDD 总结

核心优势

阶段 SDD 价值
Specify 明确功能边界，AI 提前发现需求歧义
Clarify 补充约束条件，减少实现时的返工
Plan 生成可执行的技术方案，对齐架构
Tasks 拆解细粒度任务，追踪进度
Implement AI 按规格实现，减少代码审查成本

适用场景

  • 新功能开发（需求边界清晰的）
  • 有明确验收标准（可写 Acceptance Scenarios）的任务
  • 需要与多人对齐技术方案的场景

注意事项

  1. 规格文档质量决定实现质量：Specify 阶段的描述越清晰，AI 实现越准确
  2. 数据模型必须在 Plan 前确认：如本次案例，底层数据模型理解错误导致 Plan 需要推倒重来
  3. 任务粒度要适中：64 个任务中有部分过细，实际可以合并

SDD 本质是把”需求 → 设计 → 实现”的过程显式化、结构化，让 AI 在有充分上下文的情况下生成更可靠的代码，同时也让开发者在实现前有机会发现设计问题。
